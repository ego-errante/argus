# Deterministic failure classification via preflight simulation; a local default-remedy policy stands in for the Agent

The three deterministic faults (expired blockhash, compute exceeded, bundle failure) are injected via `ARGUS_INJECT` and classified from a preflight `simulateTransaction` — the only deterministic reason-source for an all-or-nothing Jito bundle. A pure `failure::classify_failure(&SimResult) -> Option<FailureClass>` maps the cluster's error/log text to a `FailureClass`; the classified Failure is persisted, a `Remedy` is chosen by a **local default policy** that stands in for the AI Agent (Day 9-10) behind a `Policy` enum seam, and `failure::apply_remedy` executes it (refresh blockhash, raise CU limit, bump tip, hold, or abort). The whole inject → classify → remedy → resubmit loop runs inside the persisted `lifecycle_run` path (`injection_run`).

## Why

A Jito bundle is all-or-nothing, so a faulted transaction never lands and leaves **no on-chain `meta.err`** to read after submission — the post-hoc reason source other stacks rely on doesn't exist here. Preflight `simulateTransaction` is the one call that returns a deterministic `{err, logs, unitsConsumed}` reason *before* submitting, so it is the classifier's input. Simulation is gated to injected runs (`cfg.injection.is_some()`); the clean scored path stays simulation-free, paying no extra round-trip. Classification + a local remedy map is what produces the grading evidence ADR 0003 demands: three different Failure Classes must demonstrably drive three different Remedies, and the local policy makes that loop testable end-to-end before the Agent exists.

## Considered options

- **Classify from post-submission on-chain meta** — rejected: impossible for all-or-nothing bundles (nothing lands to inspect). **Jito `simulateBundle`** — not in the current SDK surface; plain RPC `simulateTransaction` suffices and is provider-neutral.
- **Expired-blockhash injection: instant-stale hash vs aged-real vs wait-out-window** — adopted **aged-real**: fetch a genuinely real blockhash from ~200 slots ago via `getBlock` (walking back over skipped slots). It is realistic (the cluster truly reports `BlockhashNotFound`, proven live), fast, and deterministic — more defensible in the README/arch-doc than a never-real hash, without the ~60-90 s wall-clock of waiting out the validity window.
- **`async-trait` for the policy seam** — rejected to keep the zero-new-crates posture (ADR 0008). A plain `enum Policy { Local, Agent(AgentClient) }` with one `async fn decide` gives the same swap-in seam; Day 9-10 replaces `Policy::Agent` with no call-site change.
- **Include `FeeTooLow` in the deterministic classifier** — rejected: fee-too-low is a probabilistic landing-contention outcome, not anything simulation reports. It stays out of the deterministic set (3 deterministic injections, per the PLAN.md cut-line); the `BumpTip` remedy is still wired for when the Agent diagnoses it.

## Consequences

`classify_failure` matches case-insensitive substrings on the combined `err` + `logs`, most-specific first — order is load-bearing because a compute error's `err` also contains `"InstructionError"` (the bundle-failure catch). **Live testing corrected the compute pattern**: the real runtime enum is `ComputationalBudgetExceeded`, not the docs' `ComputeBudgetExceeded` — both are now matched. Any unrecognized non-empty `err` falls through to `BundleFailure` (the Organic Failure catch-all). The injected attempt-1 is recorded as a real (failing) Submission row with its `failure_class` set; the decision lands in the `decisions` table (the `record_decision` sink, implemented now so Day 9-10 only swaps the decision *source*). `apply_remedy` lifts a pathological injected CU limit (1) straight to a working floor (200 k, capped at the 1.4 M per-tx max) so the remedy actually recovers in one step; `BumpTip` floors at the live p75. Abort stops with no landing (the recorded "≥2 failures" evidence). Three config-free module consts tune the remedies; one env knob (`ARGUS_INJECT`) selects the fault. Proven live (2026-06-18): all three injections classified correctly and drove `RefreshBlockhash` / `RaiseCuLimit` / `Abort` respectively; the two recoverable faults landed clean on attempt 2.

## Hardening (2026-06-18)

A `/code-review ultra` pass replaced two fragile shortcuts:

- **Classification now keys on the structured error, not flattened-string substrings.** `sim_result_from_response` parses the `InstructionError` *variant* once into `SimResult.instruction_error` (e.g. `ComputationalBudgetExceeded` out of `{"InstructionError":[idx, variant]}`); `classify_failure` matches Compute-Exceeded on that variant and treats every *other* structured instruction error as a Bundle Failure — no more substring-guessing `"instructionerror"`/`"insufficient"` against arbitrary log prose. This is the deeper fix for the same class of bug that let the real `ComputationalBudgetExceeded` enum slip past the docs-spelled `ComputeBudgetExceeded` the first time. The blockhash check stays a text match (it surfaces as a top-level message, not a structured error). The stale `ComputeBudgetExceeded` test fixture was corrected to the real runtime string.
- **`RaiseCuLimit` derives the limit from observation, not a payload-tuned constant.** A *failed* compute simulation reports `units_consumed ≈ the injected cap`, so it can't be read as the true need. The remedy path now re-simulates the clean attempt-2 transaction at the per-tx **max CU** (so the sim succeeds and `unitsConsumed` is the real requirement), then `apply_remedy` sets `cu_limit = observed × 1.5`, floored at a small minimum and capped at 1.4 M — falling back to doubling only when no observation is available. The magic 200 k floor is gone, so the remedy scales to the real payloads the Day 9+ Agent will drive.

Also: `RetryState` gained `priority_fee_microlamports` and became the single per-attempt knob carrier passed to the submit path (no more parallel loose args), and the injection run fetches its tip-floor / slot context concurrently (`tokio::join!`) off the serial path.
